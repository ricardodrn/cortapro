# Contributing / Contribuir

English below.

## Español

Gracias por tu interés en contribuir. La rama `main` está protegida: no se puede hacer push directo, force-push ni borrarla. Todo cambio entra por Pull Request.

### Cómo contribuir

1. Haz un fork del repositorio.
2. Crea una rama a partir de `main` con un nombre descriptivo:

   ```bash
   git checkout -b fix/error-en-formulario
   ```

3. Haz tus cambios con commits claros y enfocados.
4. Sube tu rama y abre un Pull Request hacia `main`, explicando qué resuelve y cómo probarlo.
5. Responde a los comentarios que surjan en la revisión. Una vez resueltos, el mantenedor hará el merge.

### Recomendaciones

- Mantén los PRs pequeños y de un solo propósito. Son más fáciles de revisar y se mergean más rápido.
- Si el cambio es grande o no estás seguro del enfoque, abre un issue primero para discutirlo. Te puede ahorrar trabajo.

### Preguntas

Abre un issue y te ayudamos a partir.

## English

Thanks for your interest in contributing. The `main` branch is protected: direct pushes, force-pushes and branch deletion are not allowed. All changes go through Pull Requests.

### How to contribute

1. Fork the repository.
2. Create a branch off `main` with a descriptive name:

   ```bash
   git checkout -b fix/form-validation-bug
   ```

3. Make your changes with clear, focused commits.
4. Push your branch and open a Pull Request against `main`, explaining what it solves and how to test it.
5. Address any review comments. Once resolved, the maintainer will merge it.

### Recommendations

- Keep PRs small and single-purpose. They're easier to review and get merged faster.
- If the change is large or you're unsure about the approach, open an issue first to discuss it. It might save you some work.

### Questions

Open an issue and we'll help you get started.
